# [DEC-006]: Supabase Auth + Storage + Edge Functions

**Date:** 2026-05-29  
**Status:** `Accepted`

## Decision

Use **Supabase Auth** for dashboard login, **Supabase Storage** (`shop-states` bucket) for per-user JSON shop state with RLS, and **Edge Functions** for Gemini AI, Telegram webhooks, and Messenger proxy.

One shop per authenticated user (MVP).

## Consequences

- Positive: No Express server; secrets stay in Edge; RLS isolates tenants
- Negative: Large bot logic ported to Deno; requires Supabase project for full stack
